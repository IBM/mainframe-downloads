IBM Explorer for z/OS Aqua

Product Name Aqua

Aqua release train version

IBM Explorer for z/OS Aqua (or simply Aqua) is a continuous delivery release train platform that brings together a set of Eclipse-based mainframe development tools and offerings in order to provide z/OS application developers and system programmers an easy way to install, deploy and maintain our IBM Eclipse-based products.

What is the difference between the versions of Aqua?

Each version of Aqua is based on a different version of Eclipse. New features and products will be integrated with newer versions of Aqua while the previous versions will remain in support mode moving forward.Aqua 3.1 is based on Eclipse 4.6 Neon, and Aqua 3.2 is based on Eclipse 4.8 Photon.

We recommend that you choose the newest version of Aqua that supports all of the products that you're looking to install since it will allow you to stay in the active development cycle longer.

Can I upgrade an Aqua environment directly to another version?

It is not possible to install plug-ins and tools onto an Aqua environment that is from a different Aqua version since each version is based on a specific Eclipse iteration. To move from one Aqua version to another (eg. Aqua 3.1 to Aqua 3.2), you will need a fresh environment and from there you can bring over the workspace data.

Client

Aqua client installation formats

IBM Explorer for z/OS Aqua (or simply Aqua) can be installed using two different packaging formats: IBM Installation Manager or Eclipse p2.

IBM Installation Manager (IM)

IBM Installation Manager is IBM’s solution for managing Eclipse-based tools such as Aqua tools. You install a separate utility (IBM Installation Manager) that keeps track of your installed Eclipse-based tools and is able to modify and extend those tools. Some Aqua tools must be installed using IBM Installation Manager — they are not available via Eclipse p2.

Eclipse p2

Eclipse p2 is an installation technology built into Eclipse tools. It can be accessed using the built-in ‘Install New Software’ option. It can often be the quickest way to get going, though not all Aqua tools are available via Eclipse p2.

What does the text under the "Client" column mean?

The text under the "Client" column indicates the available installation methods for that particular product. p2 indicates that the product can be installed via Eclipse p2, IM indicates that the product can be installed via IBM Installation Manager, and ZIP indicates that there is a pre-packaged zip file that contains the repository for installing this product in an offline environment through Installation Manager (this is only available for Aqua 3.1).
